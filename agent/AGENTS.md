# Workspace Agent Notes

You are the assistant for this Next.js workspace.

## Purpose

- Keep the repo neutral and boilerplate-oriented.
- Prefer plain starter copy over brand-specific or industry-specific language.
- Treat the app and docs as a single coherent system.

## Working Rules

1. Use the repository as the source of truth.
2. Do not invent business claims, credentials, pricing, availability, or outcomes.
3. Keep route names, metadata, navigation, and content defaults aligned.
4. When renaming a page or section, update all internal links and sitemap entries.
5. Favor concise, implementation-oriented copy over marketing language.
6. If information is missing, state the limitation clearly rather than guessing.

## Content Guidance

- Use neutral starter language for homepage, feature pages, about pages, and contact pages.
- Keep blog and placeholder content generic unless the user explicitly supplies a new domain.
- Avoid medical, clinical, patient, and doctor framing.
- Avoid brand-specific legacy names unless the user explicitly asks to restore them.

## Safety

- Do not add claims that require external validation.
- Do not imply official support, guarantees, or endorsements.
- For user-facing copy, keep the tone direct, clear, and restrained.

## Editing Habit

- Re-read affected files before editing.
- Update all references when changing routes, labels, or defaults.
- Verify the result with search and build/lint checks when practical.
